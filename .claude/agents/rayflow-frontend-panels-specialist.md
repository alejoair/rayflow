### Frontend > Paneles del editor (sidebars y footer)

- **frontend-panels#nodepalette-color-categoria-hash-hsl**: NodePalette.getCategoryColor() asigna un color de fondo por categoría de nodo usando un hash simple del string de categoría para elegir un hue de una paleta fija de 8 (baseHues) más saturación/luminosidad derivadas de bits distintos del mismo hash — puramente determinista por nombre de categoría, no configurable ni basado en metadata del backend. — evidencia: `rayflow/editor/frontend/src/components/NodePalette.tsx#getCategoryColor`
- **frontend-panels#nodepalette-doble-filtro-builtin-custom-mas-categoria**: NodePalette filtra el catálogo en dos niveles independientes de colapso: builtin/custom (builtinOpen/customOpen) y luego por categoría dentro de cada uno (Section's open state) — cada categoría tiene su propio estado de apertura no persistido (se resetea a open:true en cada montaje). — evidencia: `rayflow/editor/frontend/src/components/NodePalette.tsx#builtinOpen`
- **frontend-panels#variablespanel-parseo-json-best-effort-para-default**: VariablesPanel.parseDefault() intenta JSON.parse el valor tipeado por el usuario para el default de una variable no-str, y si falla el parseo devuelve el string crudo tal cual (sin validar contra el tipo declarado) — es posible guardar un default cuyo tipo real en runtime no coincide con variable.type. — evidencia: `rayflow/editor/frontend/src/components/VariablesPanel.tsx#parseDefault`
- **frontend-panels#custom-nodes-panel-refresh-list-silencia-errores**: CustomNodesPanel.refreshList(), handleOpen(), y handleDelete() tragan cualquier error de red/API en un catch silencioso sin mostrar feedback al usuario — solo handleCreate() propaga el error a un estado createError visible en el formulario. — evidencia: `rayflow/editor/frontend/src/components/CustomNodesPanel.tsx#refreshList`
- **frontend-panels#flowsettingsdialog-solo-edita-outputs**: FlowSettingsDialog solo edita outputs del flow (vía PinEditor) — no existe edición de 'inputs' a nivel de flow en este diálogo porque los inputs de un flow viven en los pins del nodo de entrada (OnStart, ChatTrigger, un @entry_node custom), documentado explícitamente en un comentario del propio archivo. Al guardar, valida nombres duplicados entre filas (tras trim) y bloquea el guardado mostrando la lista de duplicados; borrar todas las filas es válido y persiste un objeto outputs vacío ({}). — evidencia: `rayflow/editor/frontend/src/components/FlowSettingsDialog.tsx#Props`
- **frontend-panels#propertiespanel-prioriza-meta-del-nodo-sobre-catalogo-global**: PropertiesPanel resuelve la metadata de pins de un nodo priorizando data.meta (guardada en el nodo de React Flow al momento de crearlo) por sobre catalog[data.nodeType] (el catálogo global actual) — un nodo con pins dinámicos conserva su forma original aunque el catálogo global cambie después. — evidencia: `rayflow/editor/frontend/src/components/PropertiesPanel.tsx#meta`
- **frontend-panels#codeeditor-ctrl-s-intercepta-guardado-nativo**: CodeEditor captura Ctrl+S/Cmd+S a nivel de onKeyDown del contenedor raíz (listener nativo, no un keymap de CodeMirror) y llama preventDefault() + onSave(), evitando el diálogo nativo 'Guardar página' del navegador — único atajo de teclado en los paneles de este subsistema. El tema del editor está fijo en oneDark (@codemirror/theme-one-dark) independientemente del tema claro/oscuro del resto de la app. — evidencia: `rayflow/editor/frontend/src/components/CodeEditor.tsx#handleKeyDown`
